Never write user-facing output files to ~/Downloads. Downloads is for input files only. Put deliverables in visible project directories (e.g. the relevant `~/Documents/workspace/<project>/` folder), not Downloads and not hidden dirs.

**Why:** Nick treats Downloads as an inbox, not an output location; output buried there or in hidden dirs gets lost. (Note: some PAI skills like Art legitimately *stage* previews to ~/Downloads before copying to a project dir - that staging is fine; the rule is about final deliverables.)

**How to apply:** When producing a file Nick will use, write it to a visible project directory, or ask where he wants it. Don't leave the only copy in Downloads.
