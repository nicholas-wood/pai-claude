Nick's "daily top 5" Telegram message is generated by `~/.claude/PAI/USER/Routines/run-cadence.ts daily`, fired by the `pai-cadence-daily.timer` systemd user timer (Mon..Fri 06:30 Australia/Melbourne). The weekly review variant is `pai-cadence-weekly.timer` (Mondays 07:00). Output digests land in `~/.claude/PAI/MEMORY/STATE/cadence/{daily,weekly}-YYYY-MM-DD.md`. Telegram credentials are read from `~/.claude/.env`. The script accepts `--no-send` for dry-run testing.

**Stale-comment trap to ignore:** `~/.claude/PAI/PULSE/PULSE.toml` lines 55-57 say *"Email triage, calendar reminders, morning brief, schedule review, and intel brief are now Devi's responsibility (running on her Mac Mini)."* That's leftover boilerplate from the upstream PAI template, **not Nick's setup**. There is no Mac Mini, there is no Devi runtime, and the daily cadence runs on this Linux box. Confirmed 2026-06-09 via grep across all `~/.claude` — every "Mac Mini" / "Devi" reference is either dead boilerplate, an aspirational future-move note, or a learning-failure memory where Nick *rejected* a Mac Mini recommendation.

If something on this box looks like it must live on another machine, check the PULSE comments against the actual systemd timers (`systemctl --user list-timers`) and `crontab -l` before assuming a remote host.
