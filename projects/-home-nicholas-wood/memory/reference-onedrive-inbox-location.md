# OneDrive Inbox Location

**Path:** `~/OneDrive/00_Inbox`

This is where Nick drops material he wants me to read and act on within a session — Word docs, PDFs, email `.eml` exports, screenshots, anything he's pulled out of Outlook or another source. The `00_` prefix sorts it to the top of the OneDrive root so he can find it from any device.

It is a **transit folder, not a home**. Things sitting in it are mid-process. The OneDrive structure that holds long-term material is organised by domain (e.g. `~/OneDrive/06_Lifesaving/LSV/2026/LIMSOC/...`). When I process an inbox item I move it to its durable home; I never leave finished material in `00_Inbox`. See [[feedback-onedrive-inbox-processing-practice]] for the routine.

Pre-existing subfolder `INBOX OLD/` already lives inside `00_Inbox/` — leave it alone unless Nick asks; it is not part of the active inbox stream.

## When to look

- Whenever Nick mentions "I've put X in the inbox", "dropped this in OneDrive", "the doc/email I just shared", or any equivalent phrasing.
- At the start of any session that picks up a thread Nick was preparing to hand off — speculative `ls` is cheap; missing a dropped artefact is expensive.

## Where things tend to go from here

- LIMSOC / LSV / SLSA material → `~/OneDrive/06_Lifesaving/...` (already organised by org and year)
- AccessBuddy material → AccessBuddy workspace, not OneDrive
- General reference / archival → ask Nick if no obvious home
